================================================================================
FLASHDASH USER GUIDE
Lewco Conveyor Sorter System -- Firmware Update Tool
Version 1.30
================================================================================

OVERVIEW
--------
FlashDash is a Windows desktop application used to update firmware on the
conveyor sorter system. It manages two types of devices:

  Hub Boards (P1965)  -- The ESP32-based controller mounted in the panel.
                         One hub per sorter zone. Communicates with the PLC
                         over EtherNet/IP and with the drives over CAN bus.

  Drive Boards (P1963) -- The STM32-based motor drive boards. Up to 16 drives
                          per hub. These run the individual crossbelt motors.

FlashDash connects to hubs over your PC's Ethernet connection and sends
firmware updates over the network -- no serial cables or programming adapters needed.


================================================================================
INSTALLATION
================================================================================

STEP 1 -- Download the Installer
----------------------------------
The latest FlashDash installer is available on the project GitHub repository
under the Releases section. Download the file named:

    FlashDash-1.30-Setup.exe

STEP 2 -- Run the Installer
-----------------------------
Double-click the installer and follow the on-screen prompts. FlashDash will
be installed to Program Files and a shortcut will be added to your desktop.
Administrator privileges may be required during installation.

STEP 3 -- Install Npcap (required for hub provisioning)
---------------------------------------------------------
The "Provision Hubs" tab requires Npcap, a free network capture library.
If you plan to provision new hubs (assign IP addresses to hubs fresh from
the factory), you must install Npcap separately.

  Download Npcap from: https://npcap.com
  Run the Npcap installer and accept the default options.
  Restart your PC after installation if prompted.

If Npcap is not installed, FlashDash will still work normally for drive and
hub firmware updates -- only the Provision Hubs tab will be disabled.

STEP 4 -- Configure the Firmware Catalog (optional)
-----------------------------------------------------
If your site uses the built-in firmware catalog to download updates directly
from GitHub, go to Config > Preferences after first launch and enter the
repository URLs. See the "Firmware Catalog Setup" section at the end of this
guide for details.


================================================================================
NETWORK SETUP
================================================================================

FlashDash communicates with hub boards over Ethernet. Your PC must be on the
same IP subnet as the hubs for discovery and updates to work.

CONNECTING TO THE HUB NETWORK
-------------------------------
Connect your PC's Ethernet port to the sorter system network. This can be:
  - A direct cable from your PC to a hub's RJ45 port
  - A cable from your PC to a network switch that the hubs are also connected to

Wi-Fi will not work for hub provisioning (Provision Hubs tab). A physical
Ethernet connection is required. Wi-Fi may work for drive/hub firmware updates
on some network configurations, but a wired connection is always recommended.

SETTING A STATIC IP ADDRESS ON YOUR PC
----------------------------------------
The hub boards are assigned addresses on a fixed subnet (e.g., 192.168.1.x).
Your PC's Ethernet adapter must have an IP address on that same subnet, or
FlashDash will not be able to reach the hubs.

To set a static IP address in Windows:

  1. Open Settings > Network & Internet.
  2. Click "Ethernet" on the left, then click your Ethernet adapter.
  3. Under "IP assignment", click Edit.
  4. Change the dropdown from "Automatic (DHCP)" to "Manual".
  5. Toggle IPv4 on and enter the following:
       IP address:   192.168.1.50   (or any unused address on the hub subnet)
       Subnet mask:  255.255.255.0
       Gateway:      192.168.1.1    (or as required for your site network)
  6. Click Save.

Note: The exact subnet depends on how the hubs were provisioned for your site.
If the hubs were assigned addresses in the 192.168.1.x range, your PC must
also be in the 192.168.1.x range. Confirm the hub subnet with your system
integrator if you are unsure.

After setting the IP address, confirm the adapter link is active (the Ethernet
cable is connected and the port LED on your PC or switch is lit) before
launching FlashDash.

VERIFYING CONNECTIVITY
-----------------------
To confirm your PC can reach a hub before opening FlashDash, open a Command
Prompt (search "cmd" in the Start menu) and type:

    ping 192.168.1.101

Replace the address with the known IP of one of your hubs. If you receive
replies, connectivity is confirmed. If you see "Request timed out", recheck
your IP address settings and cable connections.


BEFORE YOU START AN UPDATE
---------------------------
1. Connect your PC to the hub network and confirm your IP address is set
   correctly (see Network Setup above).

2. Have the firmware file(s) ready, or use the built-in catalog to download
   them. Drive firmware files are named like:
       P1963-01.02.bin
   Hub firmware files are named like:
       P1965-01.02.bin

3. The PLC does not need to be stopped during a firmware update, but it is
   recommended to put the system in a safe state before flashing drives.


LAUNCHING FLASHDASH
-------------------
Double-click the FlashDash shortcut on your desktop (or find it in the Start
menu). The application will open and automatically scan the network for hub
boards. Hub boards will appear in the hub list on the left side of the Drive
Updates tab within a few seconds.

If no hubs are found automatically, click the Scan button in the hub list panel.
You can also go to Config > Network Setup to verify your network adapter
is selected correctly.


================================================================================
TAB 1: DRIVE UPDATES -- Updating Drive Board Firmware
================================================================================

Use this tab to update firmware on the P1963 drive boards (the motor drives).

STEP 1 -- Select a Hub
-----------------------
In the hub list on the left, click the hub whose drives you want to update.
The drive list in the center will populate showing all drives connected to
that hub, along with their current firmware version and status.

Drive status indicators:
  "current"   -- Drive is running the latest uploaded firmware version.
  "outdated"  -- Drive has an older firmware version. Will be included in update.
  "stuck_bl"  -- Drive is stuck in the bootloader (see Troubleshooting below).
                 Shown with an amber background. Selected automatically for recovery.
  "unknown"   -- Hub has not yet received a firmware file to compare against.

STEP 2 -- Load a Firmware File
-------------------------------
Option A: Browse for a file
  Click "Browse..." and navigate to your .bin firmware file.

Option B: Use the firmware catalog
  Click "Catalog..." to open the firmware browser. FlashDash will connect to
  the release repository and show available versions. Click a version to
  download it, then click "Use Selected". Downloaded files are cached locally
  at %APPDATA%\FlashDash\firmware_cache\ for future use (shown with a checkmark).

Once a firmware file is loaded, the version it contains will be displayed
and the "Update Drives" button will become available.

STEP 3 -- Select Drives to Update
-----------------------------------
Use the checkboxes in the drive list to select which drives to update.
The "Select All" checkbox in the column header selects or deselects all drives.
Drives flagged as "stuck_bl" are pre-selected automatically.

STEP 4 -- Update
-----------------
Click "Update Drives". FlashDash will:
  1. Upload the firmware file to the hub.
  2. Send an update command targeting your selected drives.
  3. Show a progress panel tracking each drive as it is flashed.

The hub flashes drives one at a time over the CAN bus. Depending on the number
of drives selected, this may take a few minutes. Do not close FlashDash or
disconnect the hub from power during this process.

When complete, the panel will show "Done: N updated" and the progress bar
will reach 100%. The drive list will refresh automatically showing updated
version numbers.

NOTES:
- The firmware file is uploaded to all discovered hubs simultaneously if
  multiple hubs are present, but the update is triggered per hub individually.
- If you select a new hub before uploading, you must upload the firmware file
  again (it is stored in the hub's RAM and is cleared on reboot or re-selection).


================================================================================
TAB 2: HUB FIRMWARE -- Updating Hub Board Firmware
================================================================================

Use this tab to update firmware on the P1965 hub boards (the ESP32 controllers).

STEP 1 -- Scan for Hubs
------------------------
Click "Scan Network". Hubs will appear in the table showing their name, IP
address, and current firmware version.

Hub mode indicators:
  "app"      -- Hub is running normally.
  "factory"  -- Hub is running in factory recovery mode (see Troubleshooting).
                 Row is shown in a different color and cannot be selected for
                 a normal OTA update. A "Recover Hub" button will appear instead.

STEP 2 -- Load a Hub Firmware File
------------------------------------
Click "Browse..." or "Catalog..." to select a hub firmware file (P1965-xx.xx.bin).
The process is the same as described in the Drive Updates tab above.

STEP 3 -- Select Hub(s) and Update
------------------------------------
Check the box next to the hub(s) you want to update. Click "Update Hub(s)".

FlashDash uploads the firmware to the hub, then sends an apply command.
The hub will validate the new firmware and reboot into it. This takes about
15-30 seconds. After the reboot, FlashDash will verify the new version is
running by polling the hub's status.

Rollback: If the hub detects a problem with the new firmware (e.g., version
mismatch between the file and the binary), it will automatically roll back to
the previous version. FlashDash will report this as "rolledback" in the status.

RECOVERING A HUB IN FACTORY MODE
----------------------------------
If a hub is stuck in factory mode (this can happen after a failed OTA), it will
appear in the scan results with mode "factory". To recover:
  1. Load the production hub firmware file using Browse or Catalog.
  2. Click the "Recover Hub" button that appears for that hub's row.

The factory recovery process uploads the firmware and instructs the factory app
to flash and reboot into the production image.


================================================================================
TAB 3: PROVISION HUBS -- Assigning IP Addresses to New Hubs
================================================================================

This tab is used during initial site setup to assign IP addresses to hub boards.
It requires Npcap to be installed on your PC (free download, installed separately).
If Npcap is not detected, all controls on this tab will be disabled and a
banner will prompt you to install it.

NOTE: Provisioning uses raw Layer 2 Ethernet frames and requires your PC to be
directly connected to the hub network (not through a NAT router).

STEP 1 -- Select Network Adapter
----------------------------------
Choose the Ethernet adapter connected to the hub network from the dropdown.
Only physical Ethernet adapters are shown (Wi-Fi, VPN, and virtual adapters
are filtered out automatically).

Your last-used adapter selection is remembered between sessions.

STEP 2 -- Discover Hubs
------------------------
Click "Discover Hubs". FlashDash will broadcast a discovery message and collect
responses from all hubs on the network. Each hub responds with its MAC address
and chain position (which port it was reached from).

STEP 3 -- Assign IP Addresses
-------------------------------
IP address fields will appear for each discovered hub. You can type addresses
manually, or click "Auto-fill IPs" to assign sequential addresses starting at
.101 (e.g., 192.168.1.101, 192.168.1.102) sorted by chain position.

Set the subnet mask and gateway fields as needed for your site network.

STEP 4 -- Provision
---------------------
Click "Provision". FlashDash will:
  1. Send each hub its assigned IP address individually (by MAC address).
  2. Broadcast a reboot command. All hubs reboot simultaneously (2 second delay).
  3. Verify each hub responds at its new IP address over HTTP.

After provisioning completes successfully, the site map is saved and FlashDash


BOOTP WILDCARD MODE (field provisioning)
------------------------------------------
For field technicians who need to provision hubs without prior discovery,
FlashDash offers BOOTP wildcard mode. This mode runs a BOOTP server that
assigns sequential IP addresses to any hub that boots and sends a BOOTP request.

To enable BOOTP wildcard mode:
  1. Click "BOOTP Server" to open the configuration dialog.
  2. Enter the starting IP address (e.g., 192.168.1.101).
  3. Enter the subnet mask (e.g., 255.255.255.0) and gateway.
  4. Click "Start". The BOOTP server begins listening.
  5. Power cycle the hub(s). As each hub boots, it receives the next IP.

The server automatically skips hubs that are already on the correct subnet,
avoiding unnecessary NVM writes. Hubs on a different subnet will be reassigned.

Click "Stop" when finished provisioning. The assigned addresses are logged
in the status area.
switches to the Drive Updates tab with hub information pre-loaded.


================================================================================
DEBUG MONITOR -- Viewing Real-Time Hub Logs
================================================================================

FlashDash includes a debug monitor that displays real-time logs from the hub
board. This is useful for troubleshooting CAN communication issues, monitoring
firmware updates in progress, or diagnosing hub behavior.

OPENING THE DEBUG MONITOR
--------------------------
Go to View > Debug Stream... in the menu bar. A floating dialog will open.

USING THE DEBUG MONITOR
------------------------
1. Select the hub you want to monitor from the dropdown.
2. Click "Start" to begin polling logs from the hub.
3. Logs will appear in the text area with color-coded severity levels:
     Green  = INFO messages
     Yellow = WARNING messages
     Red    = ERROR messages

The "Hide periodic status" checkbox (checked by default) filters out routine
heartbeat and status messages that would otherwise flood the display. Uncheck
it to see all messages.

CONTROLS
---------
  Start/Stop  -- Begin or stop log polling
  Copy All    -- Copy the entire log contents to clipboard
  Clear       -- Clear the log display

The debug monitor uses polling (not a persistent connection), so it will not
interfere with firmware updates or other hub operations.

If the hub reboots while monitoring, the display will show a separator line
indicating that the hub restarted and will automatically resume fetching logs.


================================================================================
TROUBLESHOOTING
================================================================================

NO HUBS FOUND
  - Confirm your PC's Ethernet adapter is on the same subnet as the hubs.
  - Go to Config > Network Setup and confirm the correct adapter is selected.
  - Confirm the hub boards are powered and their link LEDs are active.
  - Try clicking Scan or Scan Network manually.
  - Check that Windows Firewall is not blocking the application.

DRIVE STUCK IN BOOTLOADER (amber row, "stuck_bl")
  A drive may get stuck in the bootloader if it lost power during a previous
  flash attempt. It will appear in the drive list with status "stuck_bl" and
  version "BL-ONLY". FlashDash automatically selects it for recovery.
  Simply run a normal drive update with that drive selected and it will be
  recovered automatically.

UPDATE HANGS AT 0%
  This usually means the firmware file needs to be re-uploaded. Select the
  hub again in the hub list to clear the stale state, then re-upload the
  firmware file before clicking Update Drives again.

HUB SHOWS "rolledback" AFTER UPDATE
  The hub rejected the new firmware and returned to the previous version.
  This is a safety mechanism. Check that the firmware file version matches
  what you intended to install. Rebuild or re-download the firmware file
  and try again.

HUB IN FACTORY MODE
  See "Recovering a Hub in Factory Mode" in the Hub Firmware tab section above.

CATALOG BUTTON SHOWS NO RELEASES
  Check that the firmware repository URL is set correctly in
  Config > Preferences. A GitHub token may be required for private repositories
  or if the GitHub API rate limit has been exceeded.


================================================================================
FIRMWARE CATALOG SETUP (for administrators)
================================================================================

Go to Config > Preferences to configure firmware catalog access:

  Drive firmware repo URL  -- GitHub repository URL for P1963 drive firmware
  Hub firmware repo URL    -- GitHub repository URL for P1965 hub firmware
  GitHub token             -- Optional. Required for private repos or high
                              usage to avoid GitHub API rate limits.

Cached firmware files are stored at: %APPDATA%\FlashDash\firmware_cache\

Both repo URL fields default to the correct repository on a fresh install.
You should not need to change them unless directed to by American Control
Electronics.

FIRMWARE AND SOFTWARE RESOURCES
================================================================================

All firmware releases and the FlashDash installer are published to the
following GitHub repository:

  https://github.com/zzzHighNoon/APN-0202_APN-0203

WHAT IS IN THE REPOSITORY:

  Drive Board Firmware (P1963)
    Releases are tagged and named in the format:  P1963-xx.xx
    Example: P1963-01.02
    These are the firmware files for the STM32 crossbelt motor drive boards.
    Download the .bin file from the latest release under the Assets section.

  Hub Board Firmware (P1965)
    Releases are tagged and named in the format:  P1965-xx.xx
    Example: P1965-01.02
    These are the firmware files for the ESP32 hub controller boards.
    Download the .bin file from the latest release under the Assets section.

  FlashDash Installer
    The FlashDash installer (FlashDash-Setup-x.x.exe) is also published
    as a release asset in this repository.

HOW TO FIND THE LATEST RELEASE:
  1. Go to https://github.com/zzzHighNoon/APN-0202_APN-0203
  2. Click "Releases" on the right side of the page.
  3. The most recent release is shown at the top.
  4. Under "Assets", click the .bin or .exe file to download it.

NOTE: The FlashDash firmware catalog (Config > Preferences) points to this
repository by default. In most cases you can download firmware directly from
within FlashDash using the "Catalog..." button rather than visiting GitHub
manually.
